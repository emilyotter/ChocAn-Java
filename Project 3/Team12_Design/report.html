Discussion of Design:

In building our program's classes, we aimed for each class to work well on its own and with others, enhancing the program's efficiency. We ensured all parts of a class worked towards a common goal, with clear start and end points for each module. With informational cohesion, each module focused on a specific task using the same input, leading to related outcomes and a more straightforward class structure. We monitored module interactions to keep our system modular and easy to maintain, employing getter methods to avoid content coupling and keep our methods details safe. By practicing data coupling, we shared only essential simple variables between modules, reducing complexity and enhancing clarity. These strategies, together with reducing dependencies, resulted in a system that's strong, flexible, and easy to understand, where every class and module plays its part in the project's success.
